WebSocket Hub 设计 后端架构

Gateway 层用 gorilla/websocket 管理连接池，每个用户一个 ActorPool。消息全走 WS：前端发 → Hub → Actor → LLM stream → Hub.Publish → 前端渲染。SSE 仅保留服务器主动推送。
#websocket #actor更新于 2 天前

统一响应格式约定 后端架构

所有 HTTP 返回 {code: 0, data: ...} 或 {code: N, error: "..."}。前端用 unwrapJson 解包。Code 枚举：0 OK / 401 未授权 / 1101 LLM 错误 / 1102 MCP 错误。
#api #convention更新于 5 天前

PrimaryAgent 与 Conversation 1:1 绑定 后端架构

注册时自动创建 PrimaryAgent + Conversation，通过 conversation_agents 关联。Agent.id ≠ Conversation.id，前端以 conversation_id 作为 session_id。
#data-model更新于 1 周前
