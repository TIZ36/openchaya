知识库统一 · Focus 方向

~/Documents/Chaya/设计/知识库统一.md·3 分钟前★ 速记

把全屏知识库收成「极简图标栏 + 单文档画布」。中列被砍掉——它原本只是个二跳中转，却长期占着宽、又和左树职责重复。浏览需求改由临时浮层承担。

三个老问题

  • 主画布太空：稀疏卡片网格 → 单文档天然密实
  • 导航/域树乱：六类分散 → rail 六枚图标，一枚一职
  • 笔记/文档混淆 → 暖金 vs 中性的类型语言贯穿
  • 浏览→阅读跳屏 → Peek 选中即载，画布不重建

Focus 骨架

rail 60px，竖向分区：搜索 / 本地笔记 / 云端文档 / 记忆 / 知识域 / 新建。hover 出 tooltip，选中实底（不用左竖色条，遵循 app 既定偏好）。

画布永远只服务「当前这一篇」。要切换，浮层来；要写，画布在。

Peek 浮层

⌘K 或点 rail 任意类目 → 从栏缘滑出 330px 密集列表，带搜索 + 类型筛选胶囊。每行左侧类型色块（暖金=本地 / 灰=云端），右侧类型徽标。选中即载入画布、浮层淡出，画布内容原地替换不重载。

resolveWikiRef(item): local → 绝对路径（agent 可读盘） cloud → smartnoteDocuments.get（截 4000）

类型语言

同一套色彩从 rail（暖金笔记图标）→ peek（行内色块 + 徽标）→ 画布头（chip：本地·可编辑 / 云端·已同步）一以贯之。用户任何时刻都知道「这是哪种东西、能拿它做什么」。
